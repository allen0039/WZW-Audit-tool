# 审计实训万能填表助手 - 项目记忆

## 项目概述

智能审计实践教学平台（http://10.18.0.178:9350）的 Tampermonkey 油猴脚本，自动识别网页上的 SpreadJS 表格，从本地题库中匹配正确答案并逐格填写。

## 做了什么（按时间顺序）

### 1. 提取题库数据（264条）

从 `tiku/北京优度/` 下的Excel文件中提取了264个独立表格的答案数据，构建 `tiku_min.json`。

**发现的问题**：原始提取脚本遗漏了"DI"、纯字母索引号（SA、ZD等作为独立表格的情况），以及部分顶层审定表。

### 2. 生成v6油猴脚本

创建油猴主脚本，内嵌全部题库数据。功能：
- 点击右下角"🤖万能填表"按钮
- 自动识别 SpreadJS 控件（通过 GC.Spread.Sheets API）
- 搜索索引号并匹配题库
- 将答案写入对应单元格

### 3. 测试发现偏移Bug

在DA1-3、DA1-1-3等表格测试中发现v6.0有两个根本缺陷：

**Bug 1：行偏移缺失**
- 题库坐标：Excel绝对位置，r=0从表头开始
- SpreadJS模板：顶部插入了标题行+空行，实际数据从r=2开始
- 修复：引入 `calcOffset()` 函数，通过索引号的位置差动态计算偏移量

**Bug 2：Checkbox值类型错误**
- 题库数据：字符串"√"表示勾选
- SpreadJS：cellType=7的复选框需要boolean值
- 修复：检测cellType=7时，将"√"/"是"转为 `true`，其余 `false`

### 4. 发现"DI 比较信息"页面问题

用户在DI页面遇到"未检测到索引号"。

**原因**：DI页面的索引号是"DI"（纯字母，无横杠数字），正则 `/^[A-Z]{1,2}\d*(-\d+)+$/` 要求至少一个"-数字"后缀，匹配不了"DI"。

**修复**：
- 改进正则：`/^[A-Z]{1,2}\d*(?:-\d+)*$/` 兼容纯字母索引
- 扩大搜索范围：15×15 → 30×20
- 增加Sheet名回退提取
- 发现DI数据确实存在Excel中但没被提取，补上

### 5. 全量扫描所有文件

用户指出题库不完整，还有docx格式文件。重写 `extract_all.py`，扫描全部69个文件：

**扫描结果**：
- 63个xlsx → 提取出220个有索引表格 + 大量无索引子表
- 6个docx → 提取出17个表格/段落条目（询证函数据、审计报告等）
- **总计520条原始条目**（修复重复索引Bug后为532条）

**关键发现**：
- 新扫描算法把子表（FA-1, FG-2等）错误合并到父表，丢失了208条索引
- 解决方案：保留旧版题库（正确识别子表），合并新数据

### 6. 最终题库（309条有索引 + 211条参考 + 17 docx ≈ 520条，修复后532条）

- 309条有索引号 → 脚本可自动匹配填写
- 211条无索引号 → 控制测试表/内控了解表，供参考
- 17条docx → 询证函原始数据、审计报告

### 7. 检查题库覆盖完整性 & 修复重复索引Bug

用户要求检查 `extract_all.py` 是否把 tiku 下全部文件的答案都汇入 `tiku_complete.json`。

**tiku 目录结构分析**：
- 总计 **69 个文件**（63 xlsx + 6 docx），全部在 `北京优度/` 下
- 根目录 19 个文件，`舞弊、会计估计更新/` 2 个，`实质性分析程序、细节测试/` 及子目录 48 个

**两个脚本对比**：

| 脚本 | 输出文件 | 覆盖 | 问题 |
|------|----------|:----:|------|
| `extract_all.py` | `tiku_complete.json` | 68/69 | 重复索引号直接覆盖，`FA 短期借款(1).xlsx` 数据丢失 |
| `regenerate_all.py` | `tiku_all.json` | 59/69 | 仅扫 xlsx、索引匹配过严、8个旧数据残留 |

**Bug：`extract_all.py` 第226-230行**
- 两个文件有相同索引号（如 `FA 短期借款.xlsx` 和 `FA 短期借款(1).xlsx` 都是索引 FA），第二个文件会直接覆盖第一个
- 共 12 处重复索引冲突

**修复**：改为追加数字后缀（`FA` → `FA_2`），两条数据都保留。

**修复后结果**：
- 覆盖 **69/69** 文件 ✅
- 条目从 520 → **532**（12 条被覆盖的数据恢复）

## 脚本工作原理

```
用户点击按钮 → getSpread()找到SpreadJS控件
               → detectIndex()在30×20范围内搜索索引号
               → TIKU[idx]匹配题库答案
               → calcOffset()计算行/列偏移
               → 遍历答案单元格，setValue()写入
               → 支持checkbox布尔值转换
               → 显示填写结果
```

### 关键函数

- **getSpread()**: 遍历DOM找第一个 `GC.Spread.Sheets.Workbook` 实例
- **detectIndex(sheet)**: 在单元格 + Sheet名中搜索索引号（正则匹配）
- **calcOffset(sheet, idx, cells)**: 通过索引号在SpreadJS和题库中的位置差算偏移
- **autoFill()**: 主流程函数

### 注意事项

- SpreadJS控件ID固定为 `answer`（实训单据视图）
- 索引号格式：`/^[A-Z]{1,2}\d*(?:-\d+)*$/`（如DA1-3、ZA-1、DI、AB）
- 搜索范围：前30行×20列 + Sheet名回退
- 复选框(cellType=7)：题库"√" → boolean `true`
- 所有值通过 `sheet.setValue(r, c, val)` 写入
